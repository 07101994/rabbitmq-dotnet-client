Remarks

Currently, pipelining of requests is forbidden by this implementation. The AMQP 0-8 and 0-9 specifications themselves forbid pipelining, but only by the skin of their teeth and under a somewhat generous reading.

Remarks

Continuations are retrieved in FIFO order by calling Next().

In the current implementation, only one continuation can be queued up at once. Calls to Enqueue() when a continuation is already enqueued will result in NotSupportedException being thrown.

Type Description
IRpcContinuation
Remarks

It is an error to call this method when there are no waiting continuations. In the current implementation, if this happens, null will be returned (which will usually result in an immediate NullPointerException in the caller). Correct code will always arrange for a continuation to have been Enqueue()d before calling this method.
